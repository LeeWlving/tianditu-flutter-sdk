# 功能支持矩阵

## 地图与覆盖物

| 功能 | 状态 | 数据或实现来源 |
|---|---|---|
| 矢量栅格底图 | 支持 | 天地图 WMTS |
| 影像底图 | 支持 | 天地图 WMTS |
| 地形底图 | 支持 | 天地图 WMTS |
| 中文注记 | 支持 | 天地图 WMTS |
| Marker、折线、多边形、圆 | 支持 | `flutter_map` |
| 用户位置蓝点和精度圈 | 支持 | 系统定位与 `flutter_map` |
| 地图控制器和范围自适应 | 支持 | `flutter_map` |
| 矢量瓦片样式表达式 | 不支持 | 天地图 WMTS 是栅格瓦片 |
| 3D 建筑和地形 | 不支持 | 需要其他渲染引擎和数据 |
| 离线地图包 | 不支持 | 需要缓存、授权和配额设计 |

## 定位与围栏

| 功能 | 状态 | 限制 |
|---|---|---|
| 单次定位 | 支持 | 需要系统权限 |
| 持续定位 | 支持 | 应用运行期间有效 |
| 前台圆形围栏 | 支持 | SDK 本地计算 |
| 前台多边形围栏 | 支持 | SDK 本地计算 |
| 系统级后台围栏 | 不支持 | 需要 Android/iOS 原生能力 |
| 道路匹配 | 不支持 | 天地图现有接口未提供完整能力 |

| 功能 | 状态 | 服务来源 |
|---|---|---|
| 地址地理编码 | 支持 | 天地图 |
| 逆地理编码 | 支持 | 天地图 |
| 普通 POI 搜索 | 支持 | 天地图 |
| 行政区域搜索 | 支持 | 天地图 |
| 周边搜索 | 支持 | 天地图 |
| 地图视野搜索 | 支持 | 天地图 |
| 多边形搜索 | 支持 | 天地图 |
| 行政区划查询 | 支持 | 天地图 |

## 路线与导航

| 功能 | 状态 | 限制 |
|---|---|---|
| 驾车路线规划 | 支持 | 天地图 Web Service |
| 步行路线规划 | 支持 | 使用路线策略 |
| 公交请求 | 基础支持 | 返回原始 JSON，模型待完善 |
| 路线绘制 | 支持 | 路线结果转换为折线 |
| 实时导航 | 不支持 | 缺少导航状态机和道路匹配 |
| 实时路况 | 不支持 | 当前接口未接入 |
| 偏航重算 | 不支持 | 需要持续定位和导航引擎 |
| 语音播报 | 不支持 | 需要导航指令和 TTS |

## 天气

天气不属于当前天地图 Web Service SDK 的能力。需要接入独立天气供应商，并处理：

- 供应商 API Key 和域名白名单
- 经纬度到天气查询参数的转换
- 实况、预报和预警模型
- 缓存、超时、配额和降级策略
- 天气数据许可和展示署名

建议天气功能保持为业务层独立模块，不把第三方天气供应商硬编码进地图 SDK。
